# 宏观推演强化框架

本文件用于补充 `SKILL.md` 的核心流程，重点解决三个问题：如何更快识别主轴、如何给出更清晰的方向性判断、如何避免把趋势判断和时间判断混在一起。

## 1. 五层推演结构

### 第一层：识别当前宏观主轴

每次推演先判断当前驱动经济和资产价格的最大结构性力量是什么，以及它处于哪个阶段。

常见主轴包括：

- 央行利率周期：加息、暂停、降息、再通胀约束
- 美元流动性和信用周期：扩张、收缩、再融资压力、储备再配置
- 财政周期：扩张、退坡、前置发力、债务付息约束
- 地缘和贸易秩序：全球化、碎片化、关税、制裁、供应链迁移
- 债务和资产负债表周期：加杠杆、去杠杆、抵押品压力、违约风险
- 技术和产业叙事：AI 资本开支、能源转型、产业升级、生产率冲击

操作原则：

- 先判断主轴方向，再推导局部变量和资产价格。
- 局部变量通常是主轴的下游函数，除非存在强本地政策或结构性脱钩。
- 每次都要写出主轴切换信号，例如通胀重新加速、信用利差快速走阔、财政脉冲转负、汇率压力突破政策容忍区间。

### 第二层：利益结构分析

对每一个预测对象，追问各方的最优利益、约束条件和可选行动。

分析步骤：

1. 列出主要利益相关方：政府、央行、企业、居民、外资、贸易伙伴、监管机构。
2. 判断每方的约束：任期或考核目标、债务上限、通胀约束、就业压力、声誉成本、外部制裁、预算空间。
3. 推断各方在约束内最可能选择的行动。
4. 找到博弈均衡：不是预测个人意志，而是预测结构性压力下更可能出现的选择。

关键原则：

- 不预测人的心理，预测利益结构决定的行为。
- 当利益与意识形态、制度惯性或声誉约束冲突时，应把这些非经济约束显式列出。
- 有退路的一方通常更能承受短期博弈成本；没有退路的一方更依赖政策兜底或叙事管理。

### 第三层：数据锚定

数据用于锚定阶段、量级和剩余空间，不用于制造过度精确的点位预测。

| 数据类型 | 用途 | 示例 |
| --- | --- | --- |
| 存量/流量 | 判断趋势当前位置 | 库存、债务/GDP、社融存量、财政余额 |
| 速率变化 | 判断趋势加速或减速 | PMI 新订单、就业增速、库存去化速度 |
| 历史分位 | 判断定价是否极端 | 估值分位、收益率分位、利差分位 |
| 隐含预期 | 判断市场是否过度定价 | 利率期货、远期汇率、通胀预期、期权波动率 |
| 供需缺口 | 判断价格拐点 | 新增供给、库存周期、替代需求、补贴拉动 |

操作原则：

- 重点关注变化速率，而不只看绝对水平。
- 区分“方向正确”和“已经被市场充分定价”。
- 当市场定价已经透支某个逻辑时，该逻辑即使成立，也可能不再带来同方向的资产反应。

### 第四层：政策信号解读

政策不能只按表面口号理解，要看行动、预算、执行和遗漏。

解读方式：

- 加法信号：新增政策通常说明某个问题被显著提高优先级。
- 减法信号：不再提某个目标或停止某项工具，可能说明政策选择已经改变。
- 行动与表态偏差：表态与执行不一致时，以行动和资源投入为准。
- 先行动作：技术性或局部性安排可能是更大政策变化的铺垫。

自检问题：

- 这项政策解决的真实约束是什么？
- 是否有足够预算、执行主体和激励机制？
- 政策是托底、刺激、结构调整，还是预期管理？
- 市场对政策的解读是否过度乐观或过度悲观？

### 第五层：供需结构与周期位置

实体经济和商品价格判断必须回到供给、需求、库存和时间。

```text
供给端：新增供给速度 + 存量库存变化
    ↓
需求端：人口结构 + 收入预期 + 政策刺激 + 替代需求
    ↓
价格：供需缺口决定方向，金融条件决定幅度
    ↓
时间：库存周期和政策退出节奏决定拐点
```

注意边际效应衰减：

- 刺激政策常会提前透支未来需求，退出后需要观察自然需求是否接上。
- 补贴、降价和产品迭代可能制造一次性脉冲，不能直接视为长期趋势。
- 若供给仍在扩张，需求修复未必能立即带来价格反转。

## 2. 预测输出规范

当用户要求“预测”“判断”“年度十大预测”“季度展望”时，每条判断都应使用以下结构：

```text
【判断】明确的方向性结论，一句话。
【核心逻辑】1-3 条因果链，每条不超过 2 句。
【关键前提】该判断成立所需的 1-2 个必要条件。
【失效条件】哪些数据、政策或市场行为会证伪该判断。
【置信度】方向置信度：高/中/低；时间节点置信度：高/中/低。
```

### 置信度分级

| 置信度 | 适用场景 |
| --- | --- |
| 高 | 结构性趋势已经确立，数据支撑充分，政策信号明确 |
| 中 | 方向判断清晰，但时间节点、幅度或外生变量仍有较大不确定性 |
| 低 | 逻辑成立，但依赖复杂博弈、突发事件或政策选择 |

### 趋势预测与节点预测

趋势预测回答“方向是什么”，节点预测回答“何时发生”。二者必须分开标注置信度。

常见情况是方向置信度较高，但时间节点置信度较低。不要把一个高置信的趋势判断包装成高置信的时间点预测。

### 负向预测

负向预测是“某件事不会发生”或“不会在某个窗口发生”。它常常可以依靠约束条件判断，但仍必须给出失效条件。

适用场景：

- 某政策不会推出：约束来自预算、债务、通胀、制度或执行能力。
- 某资产不会单边运行：约束来自过度定价、拥挤交易或缺少增量资金。
- 某事件不会在短期发生：约束来自谈判成本、准备条件、外部压力或风险收益不匹配。

输出时可标注：`【类型：负向预测】`。

## 3. 信息收集优先级

优先级从高到低：

1. 一手数据：官方统计、央行、财政、海关、交易所、国际组织原始数据。
2. 政策文本：会议纪要、政策文件、预算文本、监管规则原文。
3. 市场定价：利率、汇率、期货、期权、信用利差、资金流。
4. 行业数据：PMI、开工率、库存、订单、运价、地产成交、产业链价格。
5. 媒体报道：只能作为信号验证和线索来源，不能作为逻辑基础。

## 4. 推演自我校验

输出前逐条检查：

- 我的结论依赖哪个关键前提？这个前提有多脆弱？
- 市场当前定价隐含了什么预期？我的判断与共识的分歧在哪里？
- 如果我错了，最可能的原因是什么？
- 方向判断和时间节点判断是否分别标注了置信度？
- 是否存在被我忽略的政策约束、预算约束、外部压力或行为激励？
- 资产含义是否只是敏感性分析，而不是个性化买卖建议？

## 5. 常见认知偏差

| 偏差 | 表现 | 修正方式 |
| --- | --- | --- |
| 利益简化偏差 | 假设所有人都按经济收益最大化行动 | 显式列出非经济约束和制度惯性 |
| 单一变量偏差 | 用一个变量解释全部资产走势 | 至少检查政策、数据、定价和资金面四类证据 |
| 鸽派或宽松偏差 | 系统性低估通胀和央行信用约束 | 单独推演通胀上行和政策不宽松情景 |
| 谈判乐观偏差 | 高估理性妥协速度 | 把摩擦成本、国内政治和声誉成本纳入模型 |
| 节点提前偏差 | 方向正确但时间过早 | 分离趋势和节点置信度，设置观察窗口 |
| 事后辩护偏差 | 预测失败后补充限定条件 | 在预测时提前写清边界和失效条件 |

## 6. 示例调用流程

用户要求年度或季度推演时，可按以下顺序输出：

1. 识别全球主轴：例如利率周期、美元流动性、财政周期或地缘重构。
2. 列出利益结构：主要政策方、企业、居民、外资和外部博弈方的约束。
3. 数据锚定：库存、利率、估值分位、信用、通胀、贸易、就业等。
4. 政策信号：新增、减少、行动与表态偏差、先行动作。
5. 供需结构：供给、需求、库存、替代需求和刺激退出。
6. 输出判断：按【判断/核心逻辑/关键前提/失效条件/置信度】格式逐条呈现。
